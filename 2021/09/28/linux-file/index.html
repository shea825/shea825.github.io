¶ 文件的基本操作

¶ open函数

  • flags取值 (可用|组合)
    • 文件打开模式
      • O_RDONLY = 0
      • O_WRONLY = 1
      • O_RDWR = 2
    • 非文件打开模式
      • O_APPEND 每次写的数据放在文件尾部
      • O_CREART
      • O_EXCL 配合O_CREART来创建文件，否则出错
  • 第三个参数取值
    多用户访问权限取值
  • fd取值
    • 文件存在时返回文件描述符
    • 通常Linux的API函数出错返回-1
      • 出错处理

¶ lseek函数

  • offset参数
    • 相对偏移量
    • 有符号数类型
类型 32位平台 64位平台
char 1个字节8位 1个字节
short 2个字节16位 2个字节
int 4个字节32位 4个字节
long 4个字节 8个字节（区别）
long long 8个字节 8个字节
point 4个字节 8个字节（区别）
  • whence参数
    • 和offset配合使用
    • whence取值
      • SEEK_SET 偏移量 = offset
      • SEEK_CUR 偏移量 = 当前文件偏移量 + offset
      • SEEK_END 偏移量 = 文件长度 + offset
  • f_offset取值
    • 函数调用成功返回文件偏移量
    • 失败返回-1

¶ read和write函数

  • buf参数
    • 指向用于保存所读出数据的缓存
    • write函数不会对buf所指向的内存有任何改动，因此加上const
  • count参数
    • 指示了需要从文件读或写出多少字节
  • rd_bytes取值
    • rd_bytes = 0 代表已读到文件末尾
  • 关于write的两个问题
    • 在文件中间某处写入数据是覆盖该处的数据
    • 当以O_APPEND标志打开文件时，可以使用lseek修改文件偏移量，但每次写都会把数据写到文件末尾

¶ close函数

  • cl_tag取值
    • cl_tag = 0 函数调用成功
  • 是否存在内存泄漏？
    • 定义 分配了内存，想释放却没办法
    • open打开或创建一个文件后，若未close，其fd还存在，随时可调用close关闭，不存在内存泄漏
